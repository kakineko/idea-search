# Outside Signal Design: Concept-Pair Temporal Analysis via arXiv + Semantic Scholar

**Date**: 2026-04-18
**Status**: Design notes, pre-implementation
**Context**: Preparing for Built with Opus 4.7 hackathon (2026-04-22 to 2026-04-27)

## Motivation

The idea-search harness currently has an internal self-evaluation layer: four LLM judges score each idea on novelty, feasibility, value, and risk. Today's experiments showed this layer is unreliable — evaluators give high scores (9.0+) to ideas that are clearly convergent on the same concept.

The problem is fundamental: when LLMs judge their own outputs, they systematically miss convergence. They score the *form* of novelty without verifying the *substance*.

So we need an external grounding signal — one that doesn't come from the same LLM distribution. That's the "outside signal".

## Core Hypothesis

Genuine novelty often looks like **concepts that only recently started being discussed in the same breath**.

Individual words in an idea might all be old (e.g., "quantum", "error", "correction"), but their *combination* may be recent (e.g., "quantum error correction" clustered around 2000 onward).

This means:
- Single-word frequency analysis is insufficient
- Full-phrase matching misses variations
- **Pair-level co-occurrence over time** is the right granularity

## Data Source Decision

### Rejected: Google Ngram Viewer

Initial design included Ngram as a long-term trend source, but rejected for two reasons:

1. **Data ends in 2019**: Most ideas generated by modern LLMs involve post-2020 concepts (LLMs, agents, RAG, etc.). Ngram cannot see these.
2. **Single source sufficient**: If we can't measure modern novelty with Ngram, it's not worth the complexity of combining two sources.

### Chosen: arXiv + Semantic Scholar

**arXiv** (1991–present):
- Excellent coverage of CS, ML, Physics
- Official REST API, Python library `arxiv`
- Rate limit: 3 seconds per request (ToS)
- Free, no authentication required

**Semantic Scholar** (1900s–present):
- 200M+ papers, broader field coverage
- Includes medicine, psychology, sociology, economics, etc.
- Official REST API, Python library `semanticscholar`
- Rate limit: 100 req/5min anonymously (~3s per request)
- API key available for higher rates

Together these cover the bias problem of arXiv-alone (weak in non-CS/Physics fields) while staying implementable in one week.

## Key Insight: Count-Only Queries

We do not need to fetch paper titles, abstracts, or full text. The outside signal only needs **yearly result counts** for each pair.

### arXiv count-only query

arXiv's Atom response includes `<opensearch:totalResults>`, which gives the total count without returning actual results.

```python
query = f'all:"{pair[0]}" AND all:"{pair[1]}" AND submittedDate:[{year}01010000 TO {year}12312359]'
url = f'http://export.arxiv.org/api/query?search_query={quote(query)}&max_results=0'
# Parse XML, extract totalResults
```

### Semantic Scholar count-only query

```python
results = sch.search_paper(
    query=f'"{pair[0]}" "{pair[1]}"',
    publication_date_or_year=f'{year}',
    limit=1
)
total = results.total
```

### Data footprint

- 1 pair × 12 years = ~12 integers = tens of bytes
- 1 idea × 10 pairs = hundreds of bytes
- 100 ideas evaluated = tens of KB

Extremely lightweight. No storage infrastructure needed.

## Open Questions (Resolve During Implementation)

### 1. Concept extraction granularity
- How many concepts per idea? (3-5? or more?)
- Single words vs multi-word phrases?
- Claude prompt needs tuning

### 2. Pair filter thresholds
- What embedding distance counts as "too close"? (< 0.3?)
- What counts as "too distant"? (> 0.8?)
- Empirical tuning required

### 3. Novelty score from time series
- Sudden increase in recent years = novel? How sudden?
- Smooth growth vs abrupt emergence — both signals?
- Candidate metrics:
  - Ratio of post-2020 count to pre-2020 count
  - Year of first non-zero entry
  - Compound annual growth rate in recent window

### 4. Aggregation across pairs
- How to combine multiple pair signals into one score?
- Average? Min? Weighted by embedding distance?

### 5. Cross-source disagreement
- What if arXiv says novel but Semantic Scholar says not?
- Field bias vs real signal: hard to distinguish automatically

## Scope Boundaries

### In scope (MVP)
- Pair-level (2 concepts) co-occurrence analysis
- arXiv + Semantic Scholar, count-only queries
- Embedding-based pair filtering
- Simple heuristic aggregation

### Out of scope (future work)
- Triple/quadruple concept combinations
- Citation graph analysis (e.g., Uzzi's atypicality)
- Field-aware data source routing (technical → arXiv, business → Crunchbase, etc.)
- Query expansion (synonyms, variants)
- Business/product data sources (Product Hunt, Crunchbase, HN Algolia)

## Related Prior Work

- **Uzzi et al., "Atypical Combinations and Scientific Impact" (Science 2013)**: measuring novelty via co-citation pair atypicality
- **Combinatorial novelty literature** (Schumpeter, recombinant innovation)
- **Conceptual blending** (Fauconnier, cognitive science)

This design adapts the Uzzi approach from citation pairs to concept pairs extracted from text, applied to ideation validation rather than post-hoc scientific measurement.

## Why This Matters

The harness has proven it can generate diverse-looking ideas (role separation works at round 1). What it cannot do is tell which of those are actually novel versus which only *sound* novel. The outside signal is the honesty layer.

It also operationalizes the thesis stated in the hackathon application:
> "An ideation harness that honestly tells you which ideas are promising vs which only sound promising — and which directions to explore before you commit."
